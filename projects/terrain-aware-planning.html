Terrain-Aware Path Planning from Point Clouds

A planning framework that converts raw 3D point clouds into traversability-aware cost maps, enabling safer navigation over varied terrain.

Point Clouds Cost Maps Mobile Robotics
View on GitHub

Problem

Traditional 2D planners assume flat terrain, which can cause unsafe or inefficient behavior in outdoor environments with slopes obstacles, or rough surfaces. The challenge is to use 3D information from LiDAR or depth sensors to inform path planning decisions.

Approach

  • Preprocess raw point clouds to remove ground plane outliers and noise.
  • Compute geometric features such as slope, roughness, and height variance in local patches.
  • Map these features into a 2.5D grid-based representation with a traversability cost for each cell.
  • Use the cost map with a standard planner (e.g., A*, D*, or sampling-based) to produce terrain-aware paths.

Results

The planner avoids highly sloped or rough regions that would be difficult for the robot to traverse, preferring more stable routes even when they are slightly longer. Visualization of the cost map and resulting paths helps interpret the planner's behavior.

Tech Stack

  • Python / C++
  • Point cloud libraries (PCL, Open3D)
  • ROS / ROS 2 integration for live planning
